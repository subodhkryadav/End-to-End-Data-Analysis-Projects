# Power BI Practice - Day 02

This practice task is based on an Office Supplies sales dataset and is focused on converting simple transactional records into a meaningful business dashboard in Power BI.

---

## Dataset Details

- **Source File:** P6-OfficeSupplies.csv
- **Supporting File:** problem Statement.png
- **Important Fields:** OrderDate, Region, Rep, Item, Units, Unit Price

The dataset captures office supply orders across regions, sales representatives, products, and pricing. It is well-suited for building summary visuals around order volume, revenue contribution, and regional performance.

---

## Project Objective

The goal of Day 02 was to practice working with a CSV file in Power BI and create a dashboard that answers basic business questions such as:

- Which region is generating stronger sales activity?
- Which representatives are contributing the most orders?
- Which items are sold most frequently?
- How can raw unit and pricing data be turned into reporting metrics?

---

## Analysis Approach

- Imported the CSV dataset into Power BI Desktop
- Reviewed the structure of date, region, rep, and product fields
- Prepared the data for visual analysis
- Used business dimensions such as Region, Rep, and Item for reporting slices
- Worked with Units and Unit Price to support sales-oriented analysis
- Followed the provided problem statement as the reporting direction

---

## Skills Practiced

- Loading flat-file data into Power BI
- Understanding dataset structure before visualization
- Creating business summaries from transactional rows
- Choosing visuals based on comparison, ranking, and distribution needs
- Building a cleaner reporting flow from problem statement to dashboard
