# Power BI Practice - Day 01

This exercise focuses on building a clean comparative sales visual in Power BI using a retail dataset from Excel. The goal was to understand how raw transactional data can be transformed into a simple but effective business view.

---

## Dataset Details

- **Source File:** P6-SuperStoreUS-2015.xlsx
- **Available Sheets:** Orders, Returns, Users
- **Primary Table Used:** Orders
- **Important Fields:** Order Date, Region, Sales, Profit, Segment, Category, Ship Mode, State or Province, City

---

## Project Objective

The main objective of this practice task was to compare sales performance across regions over time and present the output in an easy-to-read visual format.

This helped me practice:

- Importing Excel data into Power BI
- Understanding date hierarchy and time-based grouping
- Comparing multiple regions in one visual
- Creating a business-friendly chart layout

---

## Dashboard Work Completed

- Loaded the Excel workbook into Power BI Desktop
- Selected the Orders sheet as the reporting dataset
- Built a clustered bar chart for monthly regional sales comparison
- Grouped the timeline by quarter and month for better readability
- Used color legends to distinguish Central, East, South, and West regions
- Organized the chart to make cross-region comparison faster

---

## Visual Summary

- **Chart Type:** Clustered Bar Chart
- **Business Focus:** Monthly Sales Across Regions in 2015
- **X-Axis:** Quarter and Month
- **Y-Axis:** Sales
- **Legend:** Central, East, South, West
- **Tool Used:** Power BI Desktop

---

## Key Learning Outcomes

- Learned how to connect Excel-based business data to Power BI
- Practiced working with date hierarchies for time-series reporting
- Improved understanding of comparative visuals for regional analysis
- Built confidence in formatting charts for reporting clarity
